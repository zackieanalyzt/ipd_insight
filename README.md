# Healthcare CMI & NHSO Fund Transfer Dashboard (Unified Portal)

ระบบเว็บแดชบอร์ดอัจฉริยะสำหรับวิเคราะห์ประสิทธิภาพการรักษาผู้ป่วยใน (IPD CMI), ข้อมูลการรับเงินโอนจัดสรรจาก สปสช. (NHSO Fund Transfer) และการวิเคราะห์ค่าบริการทางการแพทย์ (Service Items Price Analysis) ช่วยสนับสนุนการตัดสินใจของผู้บริหารโรงพยาบาลและผู้วิเคราะห์ข้อมูลทางการแพทย์

---

## 🌟 ฟีเจอร์หลัก (Core Features)

### 1. แดชบอร์ดวิเคราะห์ประสิทธิภาพ CMI และภาระงาน (CMI Dashboard)
* **ความสัมพันธ์แบบ 3 มิติ (Bubble Chart)**: แสดงความสัมพันธ์ระหว่างปริมาณผู้ป่วยสะสม (แกน X), ค่าเฉลี่ย CMI (แกน Y) และภาระงานสะสมรวม (ขนาดของวงกลม/Bubble Size) แยกประเภทตามกลุ่มโรคหลัก (MDC)
* **กราฟแนวโน้มและการวิเคราะห์ความสัมพันธ์ (MDC Monthly Case Volume & CMI Trend)**:
  * แสดงเส้นแนวโน้มจำนวนผู้ป่วยตามเวลาแบบต่อเนื่องตามปีปฏิทินจริง
  * **กรณีเลือกเดี่ยว (Single MDC)**: ระบบจะเรนเดอร์กราฟ 2 แกน (Dual Y-Axes) แสดงความสัมพันธ์ระหว่าง **จำนวนผู้ป่วย (Volume)**, **ความซับซ้อน (CMI)** และ **ภาระงาน (AdjRW)** พร้อมกัน
  * **เส้นแนวโน้มถดถอยเชิงเส้น (Linear Regression Trend Line)**: คำนวณแบบ Least-Squares แสดงเส้นปะแนวโน้มทิศทาง พร้อม Tooltip บอกค่าความลาดชัน (Slope) ของตัวบ่งชี้ทุกตัว
  * ระบบค้นหากลุ่มโรค MDC อัจฉริยะ (Live Search Filter) ค้นได้จากรหัสและชื่อกลุ่มโรค

### 2. แดชบอร์ดรายรับเงินโอน สปสช. (NHSO Fund Transfer Dashboard)
* **สัดส่วนเงินโอนสะสมแยกตามกองทุนหลัก (Main Fund Breakdown)**: 
  * แสดงบาร์ชาร์ตเปรียบเทียบแต่ละกองทุนเรียงลำดับตามความสำคัญ
  * **ล็อกแท่ง "รวม" ไว้บนสุด**: มีแท่งยอดรวมเงินโอนของทุกกองทุนแสดงผลอยู่บนสุดเสมอ ไม่ผันผวนตามการเลือกเรียงลำดับ
  * สนับสนุนการจัดเรียง (Sorting) จากมากไปน้อย หรือน้อยไปมาก
  * คำนวณเปอร์เซ็นต์การเติบโตเปรียบเทียบปีต่อปี (YoY % Change) ใน Tooltip แบบอัตโนมัติ
* **แนวโน้มรายรับเงินโอนต่อเนื่อง (Transfer Monthly Trend)**: แสดงเส้นกราฟความเคลื่อนไหวของยอดเงินโอนรายเดือนตามปีปฏิทินต่อเนื่อง

### 3. แดชบอร์ดวิเคราะห์รายการบริการ (Service Items Dashboard)
* **สัดส่วนมูลค่าบริการแยกตามกลุ่มบริการ (Item Group Breakdown)**: แสดงกลุ่มค่าบริการหลัก 10 อันดับแรกที่โรงพยาบาลให้บริการ พร้อมระบบจัดเรียง (Sort) ตามมูลค่า
* **แนวโน้มค่าบริการรายเดือนสะสม (Price Trend)**: แสดงแนวโน้มมูลค่าบริการที่เรียกเก็บจริงตามเดือนปฏิทินต่อเนื่อง
* **ตารางแจกแจงรายการบริการ (Detailed Service Items Table)**: ค้นหาและตรวจสอบรายการบริการย่อย (Item Common Name) ได้อย่างรวดเร็ว

## ⚙️ โครงสร้างสถาปัตยกรรมและเทคโนโลยี (Architecture & Tech Stack)
* **Backend**: Node.js & Express.js ทำหน้าที่เป็น Static Web Server และ API สำรอง
* **Frontend**: Vanilla Javascript (HTML5, Custom CSS3), Chart.js (v4.x) สำหรับการพล็อตข้อมูล, PapaParse สำหรับการสตรีมและแปลงข้อมูล CSV ฝั่งเบราว์เซอร์
* **Data Layer**: ใช้ฐานข้อมูลไฟล์ CSV แบบ In-Memory ทำงานรวดเร็วสูง ไม่ต้องพึ่งพาซอฟต์แวร์ฐานข้อมูลภายนอก (SQL/PostgreSQL)

## 📂 การกำหนดค่าที่สำคัญ (Configuration File: `config.json`)

ตัวแปรและการกำหนดค่าทั้งหมดของระบบจะถูกเก็บอยู่ในไฟล์ **[config.json](file:///d:/ipd_insight/config.json)** เพื่อป้องกันการ Hardcode ในรหัสซอร์สโค้ด:

```json
{
  "server": {
    "port": 3035
  },
  "data_files": {
    "cmi": "ipd_cmi.csv",
    "transfers": "fundtransfer.csv",
    "items": "items_summary_aggregated.csv"
  }
}
```

### รายละเอียดพารามิเตอร์:
* `server.port`: พอร์ตไอพีสำหรับการเข้าถึงระบบเว็บแดชบอร์ด (ค่าเริ่มต้นคือ `3035`)
* `data_files.cmi`: ชื่อไฟล์ข้อมูล CMI (ผู้ป่วยใน)
* `data_files.transfers`: ชื่อไฟล์ข้อมูลเงินโอน สปสช.
* `data_files.items`: ชื่อไฟล์ข้อมูลสรุปรายการบริการทางการแพทย์

## 🚀 แนวทางการติดตั้งและใช้งาน (Installation & Deployment)

### 1. ความต้องการของระบบ (Prerequisites)
* ติดตั้ง **[Node.js](https://nodejs.org/)** (แนะนำเวอร์ชัน 18.x หรือใหม่กว่า) ลงบนเครื่องเซิร์ฟเวอร์หรือเครื่องคอมพิวเตอร์คอมพิวเตอร์ของคุณ

### 2. ขั้นตอนการติดตั้ง (Setup Steps)
1. ดาวน์โหลดหรือโคลนซอร์สโค้ดจากคลังข้อมูล:
   ```bash
   git clone https://github.com/zackieanalyzt/ipd_insight.git
   cd ipd_insight
   ```
2. ติดตั้งแพ็กเกจ Node.js (Dependencies):
   ```bash
   npm install
   ```
3. ตรวจสอบไฟล์แหล่งข้อมูลหลัก (.csv) ให้อยู่ในโฟลเดอร์เดียวกับโปรเจกต์:
   * ไฟล์ข้อมูล CMI (`ipd_cmi.csv`)
   * ไฟล์เงินโอน สปสช (`fundtransfer.csv`)
   * ไฟล์กลุ่มค่าบริการสรุป (`items_summary_aggregated.csv`)

### 3. การเริ่มเปิดใช้งานเซิร์ฟเวอร์ (Running the Server)
* รันคำสั่งเริ่มทำงานผ่าน Terminal:
   ```bash
   npm start
   ```
   *หรือรันโดยตรงด้วยคำสั่ง:*
   ```bash
   node server.js
   ```
* เมื่อเซิร์ฟเวอร์เริ่มทำงานแล้ว หน้าต่างจะแสดงข้อมูลพอร์ตที่เปิดใช้งาน:
   ```text
   ==================================================
    Healthcare Portal Server running at:
    http://localhost:3035/index.html
    Data Sources: CSV Files (Local In-Memory Cache)
   ==================================================
   ```

### 4. การเปิดใช้งานแดชบอร์ด
* เปิดเว็บเบราว์เซอร์ (Google Chrome, Microsoft Edge, Safari) แล้วเข้าไปที่:
  👉 **`http://localhost:3035/index.html`**

## 📊 แนวทางการบำรุงรักษาและการอัปเดตข้อมูล (Data Maintenance)

ในการอัปเดตข้อมูลให้ทันสมัยอยู่เสมอ:
1. เตรียมไฟล์ CSV ใหม่ที่มีโครงสร้างคอลัมน์เหมือนชุดข้อมูลเดิม
2. นำไฟล์ใหม่ไปวางทับในโฟลเดอร์โปรเจกต์
3. หากคุณเปลี่ยนชื่อไฟล์ CSV สามารถเข้าไปแก้ไขคีย์ชื่อไฟล์ได้ที่ไฟล์ `config.json` จากนั้นรันเซิร์ฟเวอร์ใหม่
4. **ความสดใหม่ของข้อมูล (Dynamic Refresh Badges)**: ระบบจะกวาดหาปี พ.ศ. และวันเวลาล่าสุดในไฟล์นั้นโดยอัตโนมัติ และแสดงผลเป็นป้าย `ข้อมูลล่าสุด: [วัน/เดือน/ปี]` ที่หัวมุมกราฟแต่ละจุดโดยอัตโนมัติ ช่วยลดภาระในการแก้ไขโค้ดแผงควบคุมในอนาคต
